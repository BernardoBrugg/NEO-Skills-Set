# Skill Set Disponivel

Chame uma skill com `/nome-da-skill`.

## SUPERPOWERS oficial (obra/Superpowers)

- `/using-superpowers`: bootstrap oficial; faz o agente procurar skills relevantes antes de agir.
- `/brainstorming`: transformar ideia vaga em especificacao clara.
- `/using-git-worktrees`: trabalhar em branch/worktree isolado.
- `/writing-plans`: criar plano executavel.
- `/executing-plans`: executar plano em etapas.
- `/subagent-driven-development`: dividir tarefas com subagentes e revisoes.
- `/test-driven-development`: ciclo RED-GREEN-REFACTOR.
- `/requesting-code-review`: revisar antes de continuar.
- `/receiving-code-review`: tratar feedback recebido.
- `/systematic-debugging`: investigar causa raiz.
- `/verification-before-completion`: validar antes de finalizar.
- `/finishing-a-development-branch`: finalizar branch com verificacao.

## Core

- `/confidence-check`: checar se ha contexto suficiente antes de implementar.
- `/project-skill-audit`: auditar o projeto e sugerir skills úteis.
- `/terminal-skills`: buscar e instalar novas skills públicas.
- `/github`: trabalhar com PRs, issues, CI e GitHub CLI.
- `/bmad-product-brief`: transformar ideia vaga em brief de produto.
- `/framework-best-practices`: aplicar boas práticas específicas deste framework.

## CAVEMAN

- `/caveman`: resposta curta e economia de tokens.
- `/caveman-compress`: compressão de arquivos de memória/instruções.
- `/caveman-review`: review curto e acionável.
- `/caveman-commit`: mensagem de commit curta.
- `/caveman-help`: ajuda da família CAVEMAN.
- `/caveman-stats`: estatísticas de compressão/uso.
- `/cavecrew`: orientação para subagentes com saída comprimida.

## Frontend Taste Skill (Leonxlnx/taste-skill)

- `/design-taste-frontend`: anti-slop frontend padrão para landing pages, portfolios e redesigns.
- `/gpt-taste`: variante mais rígida para GPT/Codex, com mais variação visual e motion.
- `/image-to-code`: gerar/analisar referência visual antes de codar.
- `/redesign-existing-projects`: auditar e melhorar UI existente.
- `/high-end-visual-design`: direção visual premium, agência/high-end.
- `/minimalist-ui`: UI minimalista editorial.
- `/industrial-brutalist-ui`: UI brutalista/industrial.
- `/stitch-design-taste`: regras para Google Stitch e `DESIGN.md`.
- `/full-output-enforcement`: exigir saída completa, sem placeholders.
- `/imagegen-frontend-web`: gerar referências visuais web.
- `/imagegen-frontend-mobile`: gerar referências visuais mobile.
- `/brandkit`: gerar brand kits e direção de identidade visual.
